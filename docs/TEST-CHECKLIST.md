# 카담(CADAM) 기능 테스트 체크리스트

> **용도:** 배포 전/배포 후 수동 기능 검증  
> **최종 업데이트:** 2026-03-12

## 1. Step 1~6 플로우 정상 진행

### 1-1. 차종 경로 (car path)

| # | 단계 | 확인 항목 | 통과 |
|---|------|-----------|:----:|
| 1 | Step 1 | "차종 먼저 선택할게요" 클릭 → Step 2로 이동 | ☐ |
| 2 | Step 2A | 브랜드 탭(현대/기아/제네시스) 선택 가능 | ☐ |
| 3 | Step 2A | 차종 카드 클릭 → 트림 선택 → Step 3 이동 | ☐ |
| 4 | Step 3 | 계약 기간(36/48/60개월) 선택 → Step 4 이동 | ☐ |
| 5 | Step 4 | 주행거리(1만~4만km) 선택 → Step 5 이동 | ☐ |
| 6 | Step 5 | 보증금/선납금 선택 후 "다음" 활성화 → Step 6 이동 | ☐ |
| 7 | Step 6 | 이름·연락처·동의 입력 후 제출 → `/result` 이동 | ☐ |

## 3. Step 6 폼 유효성 검사

| # | 시나리오 | 예상 결과 | 통과 |
|---|----------|-----------|:----:|
| 1 | 이름 비움 + 제출 | 인라인 에러 "이름을 입력해 주세요" | ☐ |
| 2 | 잘못된 전화번호 (예: 010-1234-567) + 제출 | 인라인 에러 "올바른 연락처를 입력해 주세요" | ☐ |
| 3 | 개인정보 동의 미체크 + 제출 | 제출 버튼 비활성 또는 에러 표시 | ☐ |
| 4 | 정상 입력 (010-1234-5678) + 동의 체크 | 제출 성공 → `/result` 이동 | ☐ |

## 4. API `/api/consultation` POST

| # | 시나리오 | 예상 결과 | 통과 |
|---|----------|-----------|:----:|
| 1 | 정상 요청 (필수 필드 모두 포함) | 200 OK, `{ success: true }` | ☐ |
| 2 | 필수 필드 누락 (name, phone 등) | 400 Bad Request | ☐ |
| 3 | 잘못된 JSON body | 400 또는 422 | ☐ |
| 4 | 24시간 내 동일 IP 중복 신청 | 409 Conflict, 경고 메시지 | ☐ |

---

## 5. Rate Limit 테스트

| # | 시나리오 | 예상 결과 | 통과 |
|---|----------|-----------|:----:|
| 1 | 동일 IP에서 60초 내 3회 이내 요청 | 정상 처리 | ☐ |
| 2 | 동일 IP에서 60초 내 4회째 요청 | 429 Too Many Requests | ☐ |
| 3 | 429 수신 시 Toast "과도한 요청입니다. 1분 후 다시 시도해 주세요." | 빨간색 Toast 표시 | ☐ |

> **참고:** Upstash Redis 환경변수 설정 필요. VPN/시크릿 모드로 IP 변경하여 테스트 가능.
