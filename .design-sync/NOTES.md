# design-sync notes — cadam-web (Rentailor UI)

This repo is a **Next.js application, not a packaged design system** — there is no
library build (`dist`), no `exports`/`main`, and no Storybook. The sync is an
**off-script, hand-scoped import** of the reusable `src/components/ui` + `icons`
subset only. Project: **Rentailor UI** (`95520e32-8fc9-422f-80e1-be88e46686d0`).

## How the build works (off-script)
- **Entry**: `.design-sync/ds-entry.tsx` — a hand-written barrel re-exporting only
  the scoped components. Passed via `--entry`. Do NOT let synth-entry run (it would
  `export *` from every src file and drag in the whole app: admin, steps, maps,
  supabase, next/navigation). `componentSrcMap` in config pins the exact set.
- **CSS**: there is no compiled CSS to scrape. `.design-sync/build-css.mjs` compiles
  the app's real `src/app/globals.css` with `@tailwindcss/postcss` (Tailwind v4),
  scoping `@source` to the synced component dirs. Output → `.ds-sync/compiled.css`,
  which `cfg.cssEntry` points at. **Re-run `node .design-sync/build-css.mjs` before
  every `package-build.mjs`.** Building from the real globals.css (not a token file)
  is deliberate: it carries the `:root` var chain that `@theme inline` references,
  all `@keyframes`, and custom utilities (`.cta-gold`, `.text-gold-gradient`, …).
- **Build command** (from repo root, after `npm i` in `.ds-sync/`):
  ```
  node .design-sync/build-css.mjs
  node .ds-sync/package-build.mjs --config .design-sync/config.json \
    --node-modules ./node_modules --entry ./.design-sync/ds-entry.tsx --out ./ds-bundle
  node .ds-sync/package-validate.mjs ./ds-bundle
  ```

## Scope decisions
- **Included**: ui = Button, FilterPill, SelectCard, ConsultationSheet, QuoteButton;
  icons = 17 standalone Icon* + LogoAnimated. All authored previews, all graded good.
- **Excluded** (intentionally — not in componentSrcMap/barrel):
  - `YoutubeModal` — pulls in `next/link` + `@/lib/car-image-url`; niche modal.
  - `ScrollToTop` — returns `null` (pure scroll behavior, no visual).
  - `GoldDefs` — invisible (`width:0 height:0`) shared-gradient `<defs>` infra.
  - `RentailorIcons` — 26-icon registry with name collisions vs standalone icons.
- Standalone icons are **self-contained**: each defines its own local gradient
  `<defs>` (e.g. `price-gold`), so they render correctly WITHOUT GoldDefs. No icon
  references the shared `gold-g1/g2/gw` ids.

## Known render warns (benign — do NOT chase on re-sync)
- `[RENDER_THIN]` on most Icon* components ("mounts paint nothing"): these are thin
  gold **line icons** — even at size 64 they paint little ink, so the area heuristic
  flags them. Confirmed visually correct on the contact sheets. Benign.

## Preview gotchas
- `ConsultationSheet` is a `position:fixed` bottom sheet. Its preview wraps it in a
  `transform: translateZ(0)` container so the fixed dim-overlay + sheet position
  relative to the card instead of escaping to the page viewport. Override:
  `cardMode: single, viewport 420x760`. Without the transform wrapper it renders
  white-on-white off-screen.
- `Button` props had to be hand-written (`cfg.dtsPropsFor.Button`) — ts-morph
  collapsed the `extends React.ButtonHTMLAttributes` interface to `[key:string]:unknown`
  in synth mode. The other components' props extracted fine.

## Fonts (Pretendard shipped)
- **Pretendard (body/sans) IS shipped.** `build-css.mjs` appends a `@font-face` +
  `:root { --font-pretendard: 'Pretendard'; }` to the compiled CSS, pointing at the
  repo's own `src/app/fonts/PretendardVariable.woff2` (variable, weight 100–900). The
  converter copies the woff2 into `fonts/`, emits `fonts/fonts.css`, and `styles.css`
  `@import`s it — so the closure carries the real face and all Korean UI text renders
  in Pretendard. Without the `:root` var, `--font-sans` (= `var(--font-pretendard), …`)
  fell back to system sans.
- **Cormorant Garamond (display) / Outfit (Latin) are NOT shipped** — deliberate: no
  synced component uses them (the `font-display`/`font-body-en` utilities aren't even
  generated by the scoped Tailwind build). They load via `next/font/google` in the real
  app. If a future scope adds components that use them, source the woff2 from
  `.next/static/media/` after a `next build` and add `@font-face` blocks the same way.
- Pretendard is Korean — never substitute a Latin font.

## Re-sync risks (what can silently go stale)
- `.ds-sync/compiled.css` is gitignored scratch — it MUST be regenerated by
  `build-css.mjs` on every sync, or the bundle ships stale/missing CSS. The script
  IS committed.
- The barrel (`ds-entry.tsx`) and `componentSrcMap` must stay in sync by hand. Adding
  a ui component requires editing both.
- `build-css.mjs` hardcodes the scoped dirs (`../components/ui`, `../components/icons`).
  If scope changes, update the SCOPED array there too.
- The thin-icon warns above are expected — an unrecorded NEW warn is the signal.
